Before you start

Claude, Hermes, and Alfred do not guarantee 100% accuracy

Treat AI outputs like a colleague's first draft: useful, worth building on, but requiring your judgment before you act. The same green / yellow / red logic that governs human escalation applies to everything AI gives you. If the output looks off, feels too confident, or touches anything consequential — apply common sense before proceeding.

Green: output makes sense — proceed Yellow: plausible but uncertain — verify first Red: touches client data or prod — confirm with a human

Ask AI first

Claude
Reasoning, research, writing, code — available to everyone
Hermes
Strategy, analysis, adaptive problem-solving
Alfred
Automated workflows and agent fleet execution at scale
Which to use: Start with whatever AI you have access to. For conversational problem-solving, research, or debugging — Claude or Hermes. For executing defined, repeatable workflows across multiple assets — Alfred. When in doubt, describe the problem to Claude first. Note: engineers cannot help troubleshoot Hermes or Alfred directly. If those agents aren't working, be prepared to fall back to Claude to accomplish the same goal a different way. Always spot-check outputs before using them in client work or passing them to another system.

Quick tips

Be creative with prompting
Think outside the box — if you're not getting what you need, ask the same question several different ways before giving up.
Come prepared when you escalate
Bring screenshots, copied text, and a summary of what you already tried. Engineers can help faster when you've done the legwork.
Run a security audit on every new tool
Before going deep on any new project or integration, ask AI to review it for privacy or compliance issues.
Make it simpler
If AI gives you a highly technical response you don't fully understand, say "explain this more simply" — keep asking until it clicks.

Role notes

Sales + Ops

AI is your first stop, not eng

  • Most analytical and research blockers AI can handle
  • Use the 30-min rule before pinging a human
  • If an output feels off, trust that instinct
SEO + Paid + Creative

AI can scale what's already working

  • Platform errors are usually config, not code — ask AI
  • Always spot-check AI output before using it in client work
  • Access issues: ops first, then eng
Everyone

Hard stops that apply to all roles

  • Any exposed credential: stop and tag eng
  • Client data somewhere unexpected: stop
  • AI output touching prod or client deliverables: human review required
